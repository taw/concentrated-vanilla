;; Description file for the event image paths
;;
;;  Syntax was simple:  choose an event from the following list and follow it with:
;;		icon  {icon type}
;;		heading_items	{no of items}		(optional)
;;		footer_items	{no of items}		(optional)
;;		importance		{high/normal}		(optional)
;;              width           {internal message scroll width | 0 - use default width} {true/false - use full scroll width (optional, deafault==false)}
;;              height          {internal message scroll height | 0 - use default height | -1 - use table height} {true/false - use full scroll height (optional, deafault==true)}
;;              framed_body     - put frame around body
;;              body_margin     {margin_width} - horizontal body margin (body_width = width - 2*body_margin).
;;		format <see below>
;;
;;
;; Where {icon type} is one of: agent, military, diplomacy, battle, historic, settlement, faction or disaster
;;
;; Heading items is the number of items that constitute the non scrolling heading part of the message.  If this isn't specified, it will default
;; to 2.  Heading items, must be single items on thier own row, otherwise the effects are undefined
;;
;; Footer items works as per heading_items, but specifies the number of non-scrolling items at the bottom of the message
;; The default is 0
;;
;; importance - messages marked as high importance will open immeadieately they appear on screen, unless more messages
;; of high importance are displayed, in which case they will open as soon as the others are closed down

;; Message formating: (not so simple)
;;
;; format <table padding>
;; {
;;	 <object type> <object_parameters>
;; }
;;
;; where <table padding> is: center, left, right, ends, all.  This affects what happens to the extra pixels on each row of the table:
;; center adds extra space to middle cell, left and right add extra to first and last cell respectively, ends adds to first and last, and all
;; divides space equally between all cells in that row.  If no table padding is specified 'all' is used by default
;; <object type> is : title, string, image, title_image, multistring (multiline_string), ui_element, group, condition, switch, columns, accept/decline, spacer
;; and <object_paramters> (per_object) are:
;;
;;		title		: row justification, font size, colour
;;		string		: row justification, font size, colour, <lookup table identifier (optional)>
;;		image		: row justification, <image name (optional)>
;;		title_image : row justification
;;		multistring : row justification, font size, colour, justification, <lookup table identifier (optional)>
;;              bodystring  : row justification, font size, colour, justification    ;; Inserts the culturally specific text retrieved from event_strings.txt
;;		ui_element  : row justification
;;		group		: spacing (distance between items in group)
;;                    {
;;						<object type> <object parameters>
;;						...
;;					  }
;;		condition   : {
;;					  true (optional)
;;					  {
;;						<object_type> <object parameters>
;;						...
;;					  }
;;					  false (optional)
;;					  {
;;						<object_type> <object parameters>
;;						...
;;					  }
;;		switch		: {
;;							case
;;							{
;;								<object_type> <object_parameters>
;;								...
;;							}
;;							case ...
;;					  }
;;		columns		: <number of columns>	font size, colour
;;		spacer		: <height>							;; insert an empty row of height <height> into the table
;;		accept		: row justification <"text">					;; Creates an "accept" button - if "text" is specified, then the button will be a text button
;;	    decline		: row justification <"text">					;; Creates an "decline" button - if "text" is specified, then the button will be a text button
;;
;;  row justification (and justification) is : center, left, right
;;	colour            : black, khaki, or rgb values
;;	font              : tnr, tnr_med, tnr_sml, verdana, verdana_med, verdana_sml (or se descr_font_db.txt) incase any new ones have been added
;;	image name        : filename of image (not in ""s, and excluding .tga extenstion)
;;	lookup identifier : id tag from message_text_lookup.txt
;;
;;
;;
;;	Specifying an optional parameter (excepting true/false in the condition object) will create a static instance of the object, i.e. it will
;;  need no further initialisation
;;
;;	The filenames will be prefixed with the path "data/ui/<culture>/eventpics", and postfixed with ".tga" so no need to worry about that.
;;	Icon type is one of: agent, military, diplomacy, battle, historic, settlement, faction or disaster.  If no icon type is specified then 'faction' is used
;;
;;  If both image and icon are specified then they should be specified in order image, icon, not icon, image.
;;
;;
;;  Message formats can be cloned by using the 'use' keyword after the format i.e.
;;
;;  settlement_taken_copy
;;		icon	settlement
;;		format  use settlement_taken
;;
;;  Unfortunately, cloned specifications will share the same title and image if the 'title', and 'title_image' message objects is used.  As 'bodystring' obejcts are reset
;;  before each use, they will be different to the version cloned
;;
